## Key Takeaways

- OWASP ZAP is an open-source web application security testing tool.
- ZAP can act as an intercepting proxy between the browser and web application.
- Automated scanning helps identify common security issues quickly.
- Manual scanning is important because tools cannot fully understand business logic.
- Authenticated scanning helps test login-protected areas of an application.
- Directory discovery helps identify exposed or hidden web paths.
- Login security requires strong controls such as MFA, rate limiting, and monitoring.
- ZAP extensions expand the tool’s functionality.
- Web application testing should always be performed in legal and authorized environments.

## Defensive Learning

- Web applications should be tested regularly for common weaknesses.
- Automated scanners should be combined with manual review.
- Authentication mechanisms should be protected with strong controls.
- Sensitive directories and files should not be exposed.
- Security headers and cookie settings should be reviewed.
- Logs should be monitored for suspicious scanning or login activity.
- Developers and security teams should work together to fix discovered issues.
- Responsible testing helps improve the security posture of web applications.
