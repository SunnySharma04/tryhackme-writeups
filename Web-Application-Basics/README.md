## Key Takeaways

- Web applications use client-server communication.
- URLs contain scheme, domain, port, path, query string, and fragment.
- HTTP requests are sent by clients to servers.
- HTTP responses are sent by servers to clients.
- HTTP methods define the action being performed.
- Headers carry important metadata.
- Status codes help understand server responses.
- Security headers help protect web applications.
- Web security requires strong understanding of HTTP basics.

## Defensive Learning

- Use HTTPS instead of HTTP.
- Validate user input properly.
- Do not expose sensitive data in URLs.
- Configure secure cookies.
- Use important security headers.
- Monitor unusual HTTP requests.
- Keep web server and backend software updated.
- Review enabled HTTP methods.
