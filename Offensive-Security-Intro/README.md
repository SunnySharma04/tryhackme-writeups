## Key Takeaways

- Offensive security means testing systems to improve security.
- Ethical hacking always requires permission.
- TryHackMe provides safe lab environments for legal cybersecurity practice.
- Thinking like an attacker helps identify security weaknesses.
- Hidden pages can expose sensitive functionality if not protected.
- Admin pages are high-value targets and must be secured properly.
- Authentication verifies identity.
- Authorization controls what a user is allowed to access.
- Security should not depend only on hidden URLs.
- Offensive security knowledge helps improve defensive security.

## Defensive Learning

- Organizations should not rely on hidden pages for security.
- Sensitive pages should always require proper authentication.
- Admin pages should be protected with strong access controls.
- Multi-Factor Authentication improves admin security.
- Login attempts should be monitored.
- Unused files and directories should be removed.
- Web applications should be tested regularly.
- Offensive security knowledge helps defenders understand attacker behavior.
- Responsible testing should only be done in authorized environments.
